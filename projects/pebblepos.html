Michelle Meksavanh (UX/UI Designer)

Order system scalable redesign and feature optimization



Client
PebblePOS has an ordering system tailored for hotel restaurants. On the visual side, the UI did not align with their brand. On the user experience side, their split payment feature and menu navigation were required excessive steps.

My Role: UX/UI Designer
Team: Aurora Data Logic, Kevin A. (Project Manager & Engineer)
Year: 2023
Timeline: 2 weeks

Goals
  • Create design library for redesign
  • Add a split payment feature
  • Reduce waiter's task time when adding menu items

The Process



Split payment

Analysis
A usability test on the old UI shows a waiter can apply multiple types of payments to a single check but to split the total evenly, the server would have to manually do the math, leaving room for human error and slowing down their workflow.

Solution
The new feature would split the total amount equally into a specified number of checks and checks are closed out in segments.

Optimize food selection

Analysis
A usability test on the old UI shows the waiter has to do a lot of back and forth to get from categories into menu items

Solution
Place all categories and items on a single screen with a quick access navigation
